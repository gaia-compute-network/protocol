# Contributing to GAIA

## Welcome

We welcome contributions from:

- **Rust/Solidity developers**: Build core protocol, smart contracts, and validation logic
- **Environmental scientists**: Design scientific reference datasets, improve oracle accuracy
- **Crypto economists**: Model tokenomics, analyze game theory, stress-test incentive alignment
- **Node operators**: Run compute and validator nodes, provide network resilience
- **Security researchers**: Audit code, identify vulnerabilities, propose hardening measures
- **Documentation specialists**: Improve clarity and accessibility of protocol documentation

## How to Contribute

### Step 1: Identify an Issue

Browse the issue tracker for work that interests you. Look for:
- Issues labeled `good-first-issue` (suitable for newcomers)
- Issues labeled `help-wanted` (community contributions encouraged)
- Issues labeled by domain (e.g., `core-protocol`, `validation`, `tokenomics`)

If you don't see an issue, feel free to open a discussion or create a new issue describing your proposed work.

### Step 2: Open a Discussion

Before starting significant work, open a discussion thread to get feedback:
- Describe your proposed change
- Explain the problem it solves or feature it adds
- Link related issues or research papers

This avoids wasted effort if the direction isn't aligned with protocol goals.

### Step 3: Submit a Pull Request

Once you have approval:
1. Fork the repository
2. Create a feature branch (e.g., `feature/freivalds-optimization`)
3. Make your changes, with clear commit messages
4. Write or update tests
5. Submit a PR with a description of changes and reference to the issue

PRs are reviewed within 7 days. Be prepared to respond to feedback and iterate.
